# 参数化BOM系统 — 产品设计文档

> 基于 InvenTree 框架，面向机械设计场景的参数化BOM管理系统
> 设计日期: 2026-06-09 | 版本: v1.0

---

## 一、产品概述

### 1.1 背景
传统 BOM 管理中，物料清单是静态的——每个装配体的零件数量和类型都是固定值。但在机械设计中：
- 同一个产品族（如辊道输送机）只是参数不同（长度、宽度、电机功率）
- 零件数量由设计参数决定（如螺栓数量 = CEIL(长度 / 200) + 2）
- 某些零件是否包含取决于参数条件（如 `速度 > 15m/min` 时才需要防护罩）

**参数化 BOM** 让 BOM 从静态清单变为动态公式系统，一套 BOM 模板通过参数配置即可生成无数具体变体。

### 1.2 目标
- 建立一套完整的参数化 BOM 管理体系，覆盖机械产品全生命周期
- 让设计人员能用公式定义 BOM 关系，而非手动填写每个数量
- 支持产品配置化——客户选参数→系统自动生成 BOM
- 兼容 InvenTree 现有数据模型，充分利用其 Part/Stock/Order 生态

### 1.3 参考系统
| 系统 | 参考要点 |
|------|---------|
| **OpenBOM** | Excel式公式引擎、xBOM多视图、多用户协同、版本控制 |
| **InvenTree** | Part参数系统、BOM继承、变体管理、已有API架构 |
| **Odoo参数化BOM(自研)** | 全局/本地参数、公式实时校验、参数库模板 |

## 二、用户画像与场景

### 2.1 角色定义
| 角色 | 职责 | 核心需求 |
|------|------|---------|
| **设计工程师** | 定义参数模板、编写BOM公式 | 灵活的参数定义、公式编写辅助、BOM预览验证 |
| **产品经理** | 规划产品族、配置产品参数 | 参数约束管理、变体生成、成本预估 |
| **生产计划员** | 根据订单生成具体BOM | 参数输入→BOM生成、物料需求计算 |
| **采购员** | 根据BOM采购 | 参数化采购清单、供应商匹配 |
| **管理员** | 系统配置、权限管理 | 参数库维护、模板发布、版本管理 |

### 2.2 核心场景

**场景A：新产品参数化定义**
1. 设计工程师创建零件分类（如"辊道输送机"）
2. 为该分类绑定参数模板（长度、宽度、速度、电机功率等）
3. 创建参数化 Part（is_template=True），挂接分类参数
4. 编写BOM公式，用参数名引用参数值
5. 发布模板

**场景B：客户配置生成BOM**
1. 销售/客户打开产品配置界面
2. 填写关键参数（长度=5000mm, 宽度=800mm, 速度=12m/min）
3. 系统根据参数计算各BOM项数量和条件
4. 生成具体BOM，展示完整物料清单和成本
5. 一键转为正式 BOM 或生成报价

**场景C：BOM变更影响分析**
1. 工程师修改某参数默认值或范围
2. 系统自动标记受影响的BOM项和产品
3. Where-used 查询可追溯所有使用该参数的产品
4. 版本对比：查看参数/公式变更前后的BOM差异

**场景D：多级参数传递**
1. 子装配体的参数由上级传递（如输送机的"总长"决定"辊道"的"有效长度"）
2. 子装配体 BOM 公式可引用传递下来的参数
3. 参数值沿 BOM 树自上而下传播，下层可覆盖

### 3.2 详细模型定义

#### 3.2.1 ParametricBomItem（参数化BOM项）
扩展 InvenTree 的 `BomItem` 模型，添加公式支持。

| 字段 | 类型 | 说明 |
|------|------|------|
| `bom_item` | OneToOne → BomItem | 关联的BOM项 |
| `qty_formula` | CharField(512) | **数量公式**。如 `CEIL(parent.length / 500) * 2` |
| `condition_formula` | CharField(512) | **条件公式**。为空=始终包含，否则公式返回true时才包含 |
| `part_selector_formula` | CharField(512) | **选件公式**。根据参数动态选择子件，如 `IF(speed>15, 'MOTOR-A', 'MOTOR-B')` |

**公式语法：**
```
# 引用参数（单级）
param.长度
param.速度

# 引用上级参数
parent.宽度

# 运算符：+ - * / ( )
# 函数：CEIL() FLOOR() ROUND() ABS() IF() AND() OR() MIN() MAX()

# 示例
CEIL(param.长度 / 500) * 2                    # 螺栓数量
IF(param.速度 > 15, param.长度 * 0.02, 0)     # 防护罩材料长度
CEIL(param.载重 / param.单辊承载)               # 辊道数量
```

#### 3.2.2 ParametricRule（参数化规则）
在产品配置过程中应用的业务规则。

| 字段 | 类型 | 说明 |
|------|------|------|
| `product_part` | FK → Part | 适用的产品模板 |
| `rule_type` | Selection | `constraint`/`calculation`/`visibility` |
| `condition_formula` | CharField(512) | 触发条件 |
| `target_param` | FK → ParameterTemplate | 目标参数 |
| `action` | CharField(64) | `set_value`/`set_min`/`set_max`/`show`/`hide` |
| `value_formula` | CharField(512) | 计算值的公式 |
| `error_message` | CharField(256) | 约束违反时的提示 |
| `priority` | Integer | 规则优先级 |
| `enabled` | Boolean | 是否启用 |

**示例规则：**
```
# 约束：当长度>8000时，必须选择加强型机架
condition: param.长度 > 8000
action: set_visibility
target: param.机架类型
value: visible

# 计算：电机功率 = 载重 * 速度 / 效率
condition: always
action: set_value
target: param.电机功率
value: param.载重 * param.速度 / 0.85
```

#### 3.2.3 PartParameterConfig（零件参数配置）
增强 InvenTree 现有的参数系统，增加配置化属性。

| 字段 | 类型 | 说明 |
|------|------|------|
| `part` | FK → Part | 关联零件 |
| `template` | FK → ParameterTemplate | 参数模板 |
| `default_value` | CharField(128) | 默认值 |
| `min_value` | Float | 最小值 |
| `max_value` | Float | 最大值 |
| `options` | JSON | 选项列表（下拉选择用） |
| `is_driving` | Boolean | 是否是驱动参数（由用户输入/配置） |
| `is_computed` | Boolean | 是否是计算参数（由公式自动计算） |
| `computation_formula` | CharField(512) | 计算公式 |
| `ui_hint` | CharField(256) | UI提示文字 |
| `display_order` | Integer | 配置界面中的显示顺序 |
| `visible_on_config` | Boolean | 在配置界面是否可见 |

#### 3.2.4 ProductConfiguration（产品配置）
一次完整的产品配置记录。

| 字段 | 类型 | 说明 |
|------|------|------|
| `template_part` | FK → Part | 基于哪个产品模板配置的 |
| `title` | CharField(128) | 配置名称 |
| `revision` | CharField(32) | 版本号 |
| `status` | Selection | `draft`/`completed`/`released`/`obsolete` |
| `created_by` | FK → User | 创建者 |
| `created_at` | DateTime | 创建时间 |
| `params_snapshot` | JSON | 参数快照（名值对，含公式中间结果） |
| `generated_bom` | JSON | 生成的BOM快照 |
| `total_cost` | Money | 计算总成本 |
| `notes` | Text | 备注 |

#### 3.2.5 ConfigParameterValue（配置参数值）
产品配置中各参数的具体值。

| 字段 | 类型 | 说明 |
|------|------|------|
| `config` | FK → ProductConfiguration | 所属配置 |
| `template` | FK → ParameterTemplate | 参数模板 |
| `value` | CharField(128) | 用户输入或计算得出的值 |
| `source` | Selection | `manual`/`computed`/`inherited`/`default` |
| `computed_at` | DateTime | 计算时间 |

## 四、公式引擎设计

### 4.1 公式语言规范

**支持的语法元素：**

| 类别 | 元素 | 示例 |
|------|------|------|
| **参数引用** | `param.参数名` | `param.长度` |
| **上级引用** | `parent.参数名` | `parent.长度` |
| **系统变量** | `sys.quantity`, `sys.level` | `sys.quantity` 基准数量 |
| **数值** | 整数、小数 | `500`, `1.5` |
| **算术** | `+ - * /` | `(a + b) * 2` |
| **函数** | `CEIL`, `FLOOR`, `ROUND`, `ABS` | `CEIL(x / 500)` |
| **逻辑** | `IF`, `AND`, `OR`, `NOT` | `IF(a>10, 1, 0)` |
| **比较** | `>`, `<`, `>=`, `<=`, `=`, `!=` | `speed > 15` |
| **聚合** | `SUM`, `COUNT`, `AVG` | `SUM(child.数量)` |

### 4.2 公式验证规则
1. 语法检查：括号匹配、函数参数个数
2. 参数解析：引用的参数必须存在于当前Part或其上级
3. 类型检查：数值运算操作数必须为数值
4. 循环引用检测：A引用B，B不能引用A
5. 预览计算：提供示例值进行试算

### 4.3 公式计算上下文
```
{
  "param": { ... },        // 当前Part的所有参数
  "parent": { ... },       // 上级Part的参数（如有）
  "children": { ... },     // 子件的聚合值
  "sys": {                 // 系统变量
    "quantity": 1,         // 本BOM的基准数量
    "level": 0,            // BOM层级
    "part_id": 123,        // 当前Part ID
    "part_name": "辊道输送机" // 当前Part名称
  }
}
```

## 七、与 InvenTree 现有功能集成

### 7.1 利用现有能力
| InvenTree 现有功能 | 在本系统中的用途 |
|-------------------|----------------|
| `PartParameter` + `ParameterTemplate` | 作为参数定义的基础存储 |
| `PartCategoryParameterTemplate` | 分类级别参数模板定义 |
| `BomItem.inherited` | 参数化BOM继承到变体 |
| `Part.is_template` + `variant_of` | 参数化模板与生成的变体关系 |
| `Part.parameters_map()` | 参数名值对的快速访问 |
| Stock / Order | 参数化BOM生成的物料可接入库存和采购 |

### 7.2 新增功能
- **公式引擎** — 全新的公式解析/计算/验证模块
- **参数配置器** — 面向用户的产品配置UI
- **动态BOM生成** — 根据参数实时计算BOM
- **规则引擎** — 约束、计算、可见性规则

### 7.3 向后兼容
- 非参数化 Part 完全不受影响
- 未设置公式的 BOM Item 行为不变
- 参数化模板不强制要求配置（默认使用静态BOM）

### 9.1 机械设计公式示例

#### 辊道输送机 BOM 公式集
```
# 辊道数量 = 输送长度 / 辊道间距
辊道数量: CEIL(param.输送长度 / 500)

# 每根辊道轴承数 = 2（两端各1）
单辊轴承数: 2
总轴承数: 辊道数量 * 2

# 螺栓数量 = 辊道数量 * 每辊螺栓数
螺栓数量: 辊道数量 * 8

需要防护罩: param.速度 > 15

# 电机功率估算
电机功率(kW): (param.载重 * param.速度 * 9.8) / (1000 * 0.85)

# 机架长度
机架长度: param.输送长度 + 1000
```

```
导轨长度: param.行程 + param.滑块长度 + 80

# 滑块数量（根据长度和载重）
滑块数量: CEIL(param.行程 / 1000) * IF(param.载重 > 500, 2, 1)

# 固定螺栓=每200mm一个+两端各2个
固定螺栓: CEIL(导轨长度 / 200) + 4
```

### 10.3 错误提示设计

所有用户可见的错误必须**清晰、可操作**，不能抛技术堆栈。

| ❌ 不好的提示 | ✅ 好的提示 |
|-------------|-----------|
| `SyntaxError: unexpected token` | ⚠️ 公式第3行第8列：`@` 符号不认识，请检查是否有拼写错误 |
| `ValidationError` | 参数"总长"的值 200 超出允许范围（500~12000） |
| `500 Internal Server Error` | 公式计算出错，已自动回退到默认值。请检查：参数是否有空值？ |
| `KeyError: xyz` | 公式中引用了 "xyz" 参数，但当前零件没有这个参数。可用参数有：长度、宽度、速度 |

## 十一、稳定性设计

### 11.1 公式安全沙箱

公式引擎必须运行在安全环境中，防止恶意代码注入：

```python
class FormulaSandbox:
    """公式安全沙箱"""

    ALLOWED_FUNCTIONS = {
        'CEIL': math.ceil, 'FLOOR': math.floor, 'ROUND': round,
        'ABS': abs, 'MIN': min, 'MAX': max,
        'IF': lambda c, t, f: t if c else f,
        'AND': lambda *a: all(a), 'OR': lambda *a: any(a),
        'NOT': lambda x: not x,
        'SUM': sum, 'COUNT': len,
    }

    # ❌ 禁止的操作
    BLOCKED_BUILTINS = {
        'exec', 'eval', 'compile', '__import__',
        'open', 'getattr', 'setattr', 'delattr',
        '__subclasses__', '__globals__', '__code__',
    }

    MAX_RECURSION_DEPTH = 20    # 防止无限递归
    MAX_EXECUTION_TIME = 0.5    # 单次计算超时500ms
    MAX_FORMULA_LENGTH = 2048   # 公式最大长度
```

**安全措施：**
1. **白名单函数** — 只允许预定义的数学/逻辑函数
2. **禁止 import/eval/exec** — 不能执行任意 Python 代码
3. **执行超时** — 单次公式计算超过 500ms 自动终止
4. **递归深度限制** — 最多 20 层嵌套
5. **公式长度限制** — 不超过 2048 字符
6. **资源隔离** — 公式进程独立运行，不共享内存

### 11.2 容错与降级

| 故障场景 | 处理方式 | 用户体验 |
|---------|---------|---------|
| 公式语法错误 | 保存时拦截，提示具体位置 | 看到红色错误标记和修复建议 |
| 运行时计算失败（如除零） | 回退到默认值/上次有效值 | 黄色警告："公式未生效，使用默认值" |
| 参数缺失 | 标记缺失参数，使用默认值 | 提示："公式引用的'xxx'参数不存在" |
| 循环引用检测 | 保存公式时实时检测 | 弹窗："检测到循环引用：A→B→C→A" |
| 引擎崩溃 | 不影响 InvenTree 核心功能 | 参数化BOM退化为静态BOM |
| 数据库回滚 | 公式变更在事务中提交 | 保存失败时回滚到编辑前状态 |

### 11.3 数据一致性保障

```sql
-- 保存 ParametricBomItem 时的约束检查
CHECK (
    -- 要么 qty_formula 不为空
    qty_formula IS NOT NULL
    -- 要么使用常规 quantity 字段（BomItem已有）
    OR EXISTS(SELECT 1 FROM bom_item WHERE id = bom_item_id)
)

-- 公式变更时自动标记 BOM 为未验证
-- 触发器：更新 parametric_bom_item → 更新 bom_item.validated = False
```

### 11.4 性能保障

#### 缓存策略
| 缓存级别 | 内容 | 失效策略 |

#### 13.3.2 一键生成变体

```
[用户确认配置] → 选择「生成变体」
    ├─ 系统创建一个新的 Part（variant_of = 模板Part）
    ├─ 参数值固化到 Parts 参数中（不再可配置）
    ├─ 所有 BOM 公式执行一次 → 结果写入标准 BomItem.quantity
    ├─ 生成的 BOM 锁定（编辑后失去参数关联）
    └─ 记录配置来源（ProductConfiguration.id）
```

**适用场景：**
- 客户订单进入生产环节，需要具体的、不可变的 BOM
- 设计定型后，不再需要参数灵活性
- 与下游 ERP/MES 系统交互时，对方需要静态 BOM

#### 13.3.3 实时展开（不生成变体）

```
[用户预览/导出] → 系统实时计算:
    ├─ 读取当前参数值
    ├─ 递归展开 BOM 树（执行各级公式）
    ├─ 返回展开结果（在内存中，不落库）
    └─ 可导出为 Excel/PDF/CSV
```

**适用场景：**
- 销售报价时快速估算成本和物料
- 设计阶段对比不同参数方案的BOM差异
- 采购询价时生成临时物料清单

### 13.4 与 InvenTree 核心流程的集成

#### 13.4.1 与 Part 管理流程集成

```
Part 生命周期中的参数化BOM调用点：

创建 Part
  │
  ├→ 普通 Part → 标准 BOM 管理（不变）
  │
  └→ 参数化模板 Part (is_parametric_template=True)
       │
       ├─ 自动绑定分类参数模板
       ├─ 初始化默认参数值
       ├─ BOM 公式编辑器激活
       └─ 保存时触发公式验证

编辑 Part
  │
  ├→ 修改参数定义 → 触发所有相关 BOM 公式重算
  ├→ 修改参数默认值 → 标记依赖的 BOM 为「待验证」
  └→ 修改公式 → 立即语法验证

复制 Part
  │
  └→ 参数定义 + 公式一起复制
       └→ 可调整为独立参数（类似 Odoo 的「复制即分叉」）
```

#### 13.4.2 与 Sales Order 流程集成

```
销售订单流程中的参数化BOM调用：

创建 Sales Order
  │
  ├→ 添加标准产品 → 标准流程
  │
  └→ 添加参数化产品
       │
       ├─ 行项目显示「待配置」标记
       ├─ 点击进入产品配置器
       │   ├─ 填写参数 → 实时预览 BOM + 成本
       │   └─ 确认配置 → 配置ID关联到订单行
       │
       └─ 配置确认后：
           ├─ BOM 展开结果随订单流转
           ├─ 成本写入订单行价格
           └─ 物料需求进入 MRP 计算

订单履行（生产/采购）：
  ├→ 如果已生成变体 Part → 使用该 Part 的标准流程
  └→ 如果未生成变体 → 展开订单时实时计算
```

#### 13.4.3 与 Build Order 流程集成

```
Build Order 中的参数化BOM调用：

创建 Build Order
  │
  ├→ 关联标准 Part → 标准流程（BOM 固定）
  │
  └→ 关联参数化模板 Part
       │
       ├─ 提示：该产品为参数化设计，需要指定配置
       ├─ 选择已有配置 / 创建新配置
       │
       ├─ 选择配置后：
       │   ├─ 展开 BOM（锁定参数值到此 Build）
       │   ├─ 计算物料需求
       │   ├─ 预留库存 / 创建采购建议
       │   └─ 生产过程中 BOM 不可变
       │
       └─ 如果不指定配置：
           ├─ 使用默认参数值展开（黄色警告）
           └─ Build 过程中参数不可再修改
```

**Build Order 状态机扩展：**
```
Build Order 状态         参数化BOM行为
──────────────────────────────────────────
Pending       → 可修改配置、重新展开BOM（未锁定物料前）
In Production → 配置锁定，BOM冻结，不可修改
Complete      → 配置固化，存档配置快照
Cancelled     → 清理未使用的配置
```

#### 13.4.4 与 Purchase Order 流程集成

```
采购流程中的参数化BOM调用：

MRP 计算
  │
  ├→ 标准物料 → 现有采购逻辑
  │
  └→ 参数化产品的物料需求
       │
       ├─ 根据每个 Sales Order 的配置分别展开 BOM
       ├─ 相同参数组合的物料合并采购需求
       └─ 采购清单中标记「使用于」的配置ID

创建 Purchase Order
  │
  └→ 采购行可追溯来源参数配置
       └─ 如供应商需要知道规格参数：

       「采购 6205 轴承 × 100 个
        用途: 订单 SO-2026-001 的 辊道输送机(长度=5000,速度=12)」
```

#### 13.4.5 与 Stock 库存流程集成

```
库存流程中的参数化BOM调用：

入库
  ├→ 标准物料 → 正常入库
  └→ 参数化产品（已生成变体）→ 入库时关联配置信息

出库（Build Order 消耗）
  ├→ 标准消耗 → 正常扣料
  └→ 参数化BOM消耗
       ├─ 根据Build Order的配置展开的BOM扣料
       └─ 如果消耗量由公式决定，记录实际消耗 vs 理论消耗

库存盘点
  └→ 参数化产品也参与盘点，不影响公式计算
```

#### 13.4.6 与 MRP（物料需求计划）流程集成

```
MRP 流程中的参数化BOM调用：

MRP 运行触发
  │
  ├→ 遍历所有在途 Sales Order
  │
  ├→ 对每个 SO 行：
  │   ├─ 如果是参数化产品 → 读取配置 → 展开BOM
  │   ├─ 汇总所有配置的物料需求
  │   └─ 相同物料合并（考虑不同参数下的用量差异）
  │
  ├→ 计算净需求（现有库存 - 在途 - 已分配 + 需求）
  │
  └→ 生成采购建议 / 生产建议

示例：
  SO-001: 辊道输送机(长度=5000) × 2
          → 展开：轴承6205×40, 螺栓M8×160, 电机0.75kW×2

  SO-002: 辊道输送机(长度=8000) × 1
          → 展开：轴承6205×32, 螺栓M8×128, 电机0.75kW×1

  汇总采购需求：
  轴承6205: 40 + 32 = 72 个
  螺栓M8: 160 + 128 = 288 个
  电机0.75kW: 2 + 1 = 3 台
```

## 十四、连续变体匹配方案（"无数种"场景）

### 14.1 问题描述

在实际机械设计中，很多零件不是有限种规格，而是**连续变化的量**：

| 场景 | 参数 | 可能的取值 | 零件数 |
|------|------|-----------|:------:|
| 货架立柱 | 高度 | 1000~6000mm，每1mm一档 | **5000种** |
| 输送辊道 | 长度 | 300~2000mm，每5mm一档 | **340种** |
| 板材切割 | 长×宽 | 任意尺寸组合 | **无限种** |
| 型材/管材 | 长度 | 按需裁切 | **无限种** |

**传统做法存在的问题：**
- ❌ 给每个规格建一个 Part → 不可行（5000个立柱Part？）
- ❌ 只建几个典型规格 → 客户要非标时就无法匹配
- ❌ BOM里写死规格 → 没有追溯性，不便于采购和生产

### 14.2 三层解决方案

针对"连续变体"场景，参数化BOM提供**三种互补策略**，按需组合使用：

```
                      无限变体零件
                            │
            ┌───────────────┼───────────────┐
            ▼               ▼               ▼
      ① 规则匹配       ② 参数化生成      ③ 规格描述
     (离散映射)       (动态创建)       (生产指令)
            │               │               │
     ┌──────┴──────┐  ┌────┴────┐    ┌────┴────┐
     │预创建N个典型 │  │模板Part+  │   │BOM中写规格│
     │选型规则映射  │  │按需生成  │   │不创建Part │
     └─────────────┘  └─────────┘    └─────────┘
```

### 14.3 方案一：规则匹配（离散映射）

#### 原理
预创建一批**典型的离散规格**Part，用规则将连续参数映射到最合适的那个。适用于：参数变化对成本和结构有阶梯性影响的场景。

```
货架高度 → 立柱型号映射规则：

IF param.高度 <= 1500  THEN '立柱-1500'     # Part#201
IF param.高度 > 1500 AND param.高度 <= 2500 THEN '立柱-2500'  # Part#202
IF param.高度 > 2500 AND param.高度 <= 3500 THEN '立柱-3500'  # Part#203
IF param.高度 > 3500 AND param.高度 <= 4500 THEN '立柱-4500'  # Part#204
IF param.高度 > 4500                        THEN '立柱-4500+加强筋' # Part#205
```

#### 数据模型

```python
class PartSelectionRule(models.Model):
    """零件选择规则 - 连续参数到离散零件的映射"""
    bom_item = models.ForeignKey(ParametricBomItem, on_delete=models.CASCADE)
    name = models.CharField('规则名称', max_length=128)
    priority = models.IntegerField('优先级', default=0)
    enabled = models.BooleanField('启用', default=True)

    # 规则条件（基于参数表达式）
    condition_formula = models.CharField('条件公式', max_length=512)
    # 如: "param.高度 > 1500 AND param.高度 <= 2500"

    # 匹配结果
    target_part = models.ForeignKey(Part, on_delete=models.SET_NULL, null=True)
    # 或：没有对应Part时，直接输出规格描述
    specification = models.CharField('规格描述', max_length=256, blank=True)
    """当 target_part 为空时，用此文本描述，如 '立柱-非标H2800'"""

    # 该规则下的BOM数量公式
    quantity_formula = models.CharField('数量公式', max_length=512, default='1')
```

#### 适用场景
- 参数变化有明显的**经济分档**（如电机按功率分档：0.75kW/1.5kW/3.0kW）
- 零件有**标准系列**（如型材规格、轴承型号）
- 不想在系统里建太多 Part，但又需要**库存可追踪**

#### 优缺点
| 优点 | 缺点 |
|------|------|
| 保留库存追踪能力 | 非标尺寸无法精确匹配 |
| Part数量可控（N=5~20） | 需要预定义规则 |
| 采购和生产流程标准 | 规则维护有成本 |

### 14.4 方案二：参数化生成（动态创建）

#### 原理
创建一个**参数化模板 Part**（如 `立柱-参数化`），配置确认时**按需生成具体变体 Part**（如 `立柱-H2800`）。适用于：参数变化连续、客户定制化程度高、每个配置都可能不同的场景。

```
场景：货架高度=2800mm

模板 Part: "立柱-参数化" (Part#300, is_parametric_template=True)
  ├─ 参数: 高度=?, 材质=?, 表面处理=?
  └─ BOM公式引用这些参数

配置确认时 → 系统动态创建具体Part:
  Part#301: "立柱-H2800" (variant_of=Part#300)
  ├─ 参数: 高度=2800, 材质=Q235, 表面处理=喷塑
  ├─ 作为具体Part入库
  └─ 价格/成本根据高度公式计算
```

#### BOM公式引用参数化Part

```
在货架的BOM中：
┌─ 横梁: Part#310 × 4
├─ 层板: Part#311 × CEIL(param.层数)
├─ 立柱: Part#300 (参数化模板) × 2
│       ↑ 这里的 Part#300 是参数化模板
│       ↑ 配置时传入参数：高度=param.货架高度
│       ↑ 展开时自动创建"立柱-H2800"这个变体
│       ↑ 其BOM也根据参数展开
└─ 螺栓: Part#312 × CEIL(param.层数 * 8)
```

#### 系统行为

```python
# 配置展开时，识别到BOM中引用的是参数化模板Part
# 自动执行：

def resolve_parametric_part(template_part, params):
    """
    解析参数化模板Part为具体变体。

    逻辑：
    1. 检查是否已存在相同参数的具体Part
       - 存在 → 复用（减少Part重复）
       - 不存在 → 创建新的变体Part
    2. 生成的Part命名规范：
       {模板名}-{关键参数缩写}
       例: "立柱-H2800"
    3. 参数值固化到新Part的参数中
    4. 新Part的BOM根据公式展开并固化
    """
    # 生成唯一标识
    param_sig = generate_param_signature(template_part, params)

    # 查缓存
    existing = Part.objects.filter(
        variant_of=template_part,
        barcode_hash=param_sig
    ).first()

    if existing:
        return existing

    # 创建新变体
    new_part = Part(
        name=f"{template_part.name}-H{params['高度']}",
        variant_of=template_part,
        description=f"{template_part.description} 高度={params['高度']}mm",
        category=template_part.category,
        ...
    )
    new_part.save()

    # 固化参数
    for k, v in params.items():
        set_part_parameter(new_part, k, v)

    # 展开并固化BOM
    expand_and_lock_bom(new_part, params)
    return new_part
```

#### 参数签名缓存
为避免重复生成相同的 Part（多次配置都选H2800），使用参数签名：

```python
def generate_param_signature(template_part, params):
    """生成参数签名，用于唯一标识一个参数组合"""
    import hashlib, json
    sorted_params = json.dumps(params, sort_keys=True)
    raw = f"{template_part.id}:{sorted_params}"
    return hashlib.sha256(raw.encode()).hexdigest()[:16]
```

#### 适用场景
- 客户定制化程度高，**每个项目都可能不同**
- 零件规格是**连续值**（如任意长度的管材、任意高度的立柱）
- 希望保留**库存追踪和成本核算**能力

#### 优缺点
| 优点 | 缺点 |
|------|------|
| 覆盖所有可能的规格 | 生成的 Part 数量会增长 |
| 保留完整库存和成本追踪 | 同一规格多次配置会产生多个Part（可通过缓存避免） |
| 自动命名，无需人工干预 | 需要管理参数化模板的分类 |

### 14.5 方案三：规格描述（生产指令）

#### 原理
BOM 中的某些项不关联具体 Part，而是描述一个**生产规格**。适用于：按单生产的定制件、外协加工件、原材料切割。

```
┌─ 货架BOM (普通Part) ────────────────────────────────┐
│                                                      │
│ 零件              数量   类型          来源          │
│ ─────────────────────────────────────────────────── │
│ 横梁-Part#310      4     标准Part      仓库发料      │
│ 立柱                2     ★规格描述    外协加工      │
│                            └─ 材质: Q235             │
│                            └─ 高度: 2800mm           │
│                            └─ 截面: 80×40×3.0方管   │
│                            └─ 表面: 喷塑 RAL9016    │
│ 螺栓-Part#312      16    标准Part      仓库发料      │
│ ─────────────────────────────────────────────────── │
│ 注：立柱为定制件，规格见订单附图的 DWG-2026-005     │
└──────────────────────────────────────────────────────┘
```

#### 数据模型

```python
class BomSpecification(models.Model):
    """BOM规格描述 - 用于不需要创建Part的定制件"""
    bom_item = models.ForeignKey(BomItem, on_delete=models.CASCADE)
    specification_type = models.SelectionField([
        ('fabrication', '自制件'),
        ('outsource', '外协件'),
        ('raw_material', '原材料'),
        ('custom', '客户提供'),
    ], default='fabrication')

    # 规格参数（JSON格式，可根据模板定义）
    spec_data = models.JSONField('规格参数')
    """
    示例：
    {
        "material": "Q235",
        "dimensions": {"length": 2800, "width": 80, "height": 40},
        "wall_thickness": 3.0,
        "surface_treatment": "喷塑 RAL9016",
        "tolerance": "±0.5",
        "drawing_ref": "DWG-2026-005",
        "notes": "两端开安装孔，详见图纸"
    }
    """

    # 规格模板（可选）
    spec_template = models.ForeignKey('SpecTemplate', on_delete=models.SET_NULL, null=True)
    """预设规格模板，如'方管立柱'、'钢板底座'等"""

    unit_cost_formula = models.CharField('单价公式', max_length=512, blank=True)
    """如: param.高度 * 0.035 + 25  (每mm材料费0.035元+加工费25元)"""
```

#### 适用场景
- **非标定制件** — 每个项目尺寸都不同
- **外协加工件** — 不进入本厂库存，直接由供应商生产
- **原材料** — 板材/管材按需裁切，不维护单个Part
- **样品/试制** — 还没定型的产品

#### 优缺点
| 优点 | 缺点 |
|------|------|
| 不需要创建Part，零维护成本 | 没有库存追踪 |
| 灵活支持任意规格 | 无法复用已生产的零件 |
| 直接对接生产图纸和技术要求 | 成本估算依赖公式，精度有限 |

### 14.6 三种方案对比与选择指南

| 维度 | ①规则匹配 | ②参数化生成 | ③规格描述 |
|------|:---------:|:----------:|:---------:|
| **Part数量** | 少（固定） | 随使用增长 | 0 |
| **库存追踪** | ✅ 完整 | ✅ 完整 | ❌ 无 |
| **覆盖任意规格** | ❌ 阶梯覆盖 | ✅ 任意值 | ✅ 任意值 |
| **采购流程支持** | ✅ 标准采购 | ✅ 标准采购 | ⚠️ 按图采购 |
| **成本核算** | ✅ 精确 | ✅ 公式估算 | ⚠️ 公式估算 |
| **复用性** | ✅ 同规格复用 | ✅ 同规格复用 | ❌ 每次都新加工 |
| **适用场景** | 标准系列产品 | 半定制产品 | 完全定制产品 |

### 14.7 三种方案可以自由组合

同一个 BOM 里可以混用多种方案：

```
货架 高度=2800, 层数=4, 载重=500

BOM展开：
├─ 立柱 × 4                      ← 方案② 参数化生成 "立柱-H2800"
│  (由模板 Part#300 生成变体)
│
├─ 横梁 × 8                      ← 方案① 规则匹配 (标准件)

│
├─ 层板 × 4                      ← 标准 Part (固定BOM)
│  Part#312 '层板-标准'
│

│  Part#313 '螺栓连接件'
│
└─ 立柱底板 × 4                   ← 方案③ 规格描述 "定制底板"
```

### 14.8 对选件公式的扩展

回到最开始的**选件公式（Part Selector Formula）**，现在我们要扩展它支持三种模式：

```python
# 模式1：直接选具体Part（固定选择）
part_selector = "IF(param.载重>500, 'Part#311', 'Part#310')"

# 模式2：选参数化模板 + 传参（动态生成）
part_selector = {
    "template_part_id": 300,      # 立柱-参数化 模板 Part ID
    "params": {
        "高度": "param.货架高度",   # 从上级参数传入

}

# 模式3：输出规格描述（不选Part，直接写规格）
part_selector = {
    "spec_type": "outsource",
    "spec_template": "方管立柱",
    "spec_params": {
        "长度": "param.货架高度 + 100",
        "截面": "'80×40×3.0'",
        "表面": "'喷塑 RAL9016'"
    },
    "unit_cost": "param.货架高度 * 0.035 + 25"
} 
```
| 术语 | 英文 | 定义 |
|------|------|------|
| 参数化BOM | Parametric BOM | 用公式和参数定义的动态BOM |
| 驱动参数 | Driving Parameter | 由用户输入的关键设计参数 |
| 计算参数 | Computed Parameter | 由公式根据驱动参数计算得出的参数 |
| 条件公式 | Condition Formula | 控制BOM项是否包含的布尔公式 |
| 选件公式 | Part Selector Formula | 根据参数动态选择具体零部件的公式 |
| 配置快照 | Configuration Snapshot | 某次配置的参数值和BOM的完整记录 |
| BOM模板 | BOM Template | 含有公式的参数化BOM定义 |
| 规则匹配 | Rule-based Selection | 用规则将连续参数映射到离散Part，适用于标准系列产品 |
| 参数化生成 | Parametric Generation | 从模板Part按需生成具体变体，适用于半定制产品 |
| 规格描述 | Specification-based BOM | BOM中直接写加工规格不创建Part，适用于完全定制产品 |

### 15.3 场景详解与示例

---

#### 场景1：控制数量 🎯

**核心问题：** 做1个成品，某个零件需要多少个？

**公式示例：**
```
# 辊道数量 = 输送长度 ÷ 间距（向上取整）
CEIL(param.输送长度 / 500)

# 螺栓数量 = 连接长度 ÷ 螺栓间距 × 排数
CEIL(param.连接长度 / 200) * 2

# 加强筋数量
IF(param.长度 > 3000, CEIL(param.长度 / 1000), 0)

# 材料用量 = 长度 × 宽度 × 厚度 × 密度 / 1000000 (kg)
param.长度 * param.宽度 * 5 * 7.85 / 1000000
```

**适用零件：** 辊道、螺栓、轴承、加强筋、材料用量

---

#### 场景2：条件包含 ⚡

**核心问题：** 这个零件要不要出现在BOM里？

**公式示例：**
```
# 防护罩：速度超过15m/min才需要
param.速度 > 15

# 加强机架：载重超过2000kg才需要
param.载重 > 2000

# 加热装置：只在北方地区需要
param.地区 == '北方'

# 备选方案：大订单才配双电机
param.长度 > 10000 AND param.速度 > 20
```

**适用零件：** 防护罩、加强件、选装配件、区域差异化零件

---

#### 场景3：选择零件 🔀

**核心问题：** 有好几个类似零件可选，根据参数自动选最合适的那个。

**公式示例：**
```
# 电机选型：轻载用小电机，重载用大电机
IF(param.载重 <= 1000, 'Part#11_电机-0.75kW', 'Part#12_电机-1.5kW')

# 轴承选型：速度高用精密轴承
IF(param.转速 > 3000, 'Part#20_精密轴承', 'Part#21_普通轴承')

# 材质选择：腐蚀环境用不锈钢
IF(param.环境 == '腐蚀', 'Part#30_不锈钢螺栓', 'Part#31_镀锌螺栓')

# 多条件嵌套：综合判断
IF(
  AND(param.载重 > 2000, param.速度 > 15),
  'Part#40_加强型机架',
  IF(param.载重 > 1000, 'Part#41_标准机架', 'Part#42_轻型机架')
)
```

**适用零件：** 电机、轴承、螺栓等级、材质选择、表面处理方式

---

#### 场景4：动态生成变体 🏭

**核心问题：** 零件有无穷多种规格，无法预建所有 Part。

**配置示例：**
```
引用方式：在 BOM 中引用参数化模板 Part

BOM项配置：
{
  "template_part_id": 300,           // "立柱-参数化" 模板
  "params": {
    "高度": "param.货架高度",          // 从货架的"高度"参数传入
    "材质": "'Q235'",                 // 固定值
    "表面处理": "'喷塑 RAL9016'"
  }
}

配置高度=2800 时：
→ 系统查找是否已有 立柱-H2800
   └─ 没有 → 创建 Part#301 "立柱-H2800" (variant_of=300)
            → 参数固化：高度=2800
            → BOM公式展开并锁定

配置高度=3200 时：
→ 系统查找是否已有 立柱-H3200
   └─ 没有 → 创建 Part#302 "立柱-H3200"
```

**适用零件：** 任意长度的型材、任意高度的立柱、任意尺寸的板材、非标管材

#### 场景5：规格描述 📝

**核心问题：** 这个零件不进入库存，直接让车间或供应商按图加工。

**配置示例：**
```
BOM项配置（选件公式的第三种模式）：
{
  "spec_type": "outsource",          // 外协件
  "spec_template": "方管立柱",        // 规格模板

    "长度": "param.货架高度 + 100",

    "表面处理": "'喷塑 RAL9016'"
  },
  "drawing_ref": "'DWG-2026-' + param.订单号"  // 图纸号自动生成

展开结果 → BOM中的表现：
┌─────────────────────────────────────────────────────────┐

│  └─ 表面: 喷塑 RAL9016, 图纸: DWG-2026-005            │
```

**适用零件：** 定制钣金件、外协焊接件、非标机加工件、原材料裁切

#### 场景6：选择供应商 🏢

**核心问题：** 根据订单参数，自动决定向哪家供应商采购。

**配置示例：**
```
# 大批量用价格便宜的供应商，小批量用交货快的
IF(param.数量 > 1000, '供应商A_批量价', '供应商B_零售价')

# 特殊材质指定供应商
IF(param.材质 == '不锈钢316', '供应商C_特种材料', '供应商D_普通材料')

# 按地区就近采购
region_supplier(param.交货地区)
```

**适用零件：** 通用原材料、标准件、外协加工服务

**数据模型扩展：**

```python
class SupplierSelectionRule(models.Model):

    priority = models.IntegerField(default=0)
```

#### 场景7：计算参数 🧮

**核心问题：** 某个参数的值不是用户输入的，而是由其他参数算出来的。

**公式示例：**
```
# 电机功率 = 载重 × 速度 × 9.8 / (1000 × 效率)
param.载重 * param.速度 * 9.8 / (1000 * 0.85)

# 总重 = 各部件重量之和
param.辊道总重 + param.机架重 + param.电机重

# 皮带长度 = 2×中心距 + π×(D1+D2)/2 + (D2-D1)²/(4×中心距)
2 * param.中心距 + 3.14159 * (param.带轮1 + param.带轮2) / 2 + \
  (param.带轮2 - param.带轮1)**2 / (4 * param.中心距)

# 价格 = 基价 + 每米加价 × 长度
2500 + 150 * param.长度
```

**适用场景：** 工程设计计算、成本估算、重量/体积计算

#### 场景8：参数约束 🔒

**核心问题：** 某些参数不能任意选，受其他参数约束。

**规则示例：**
```
规则1：长度 > 8000 时，必须选择加强型机架（约束规则）
  IF param.长度 > 8000 → 机架类型.选项 = ['加强型']  (隐藏'标准型')

规则2：载重和速度不能同时取最大值（冲突约束）
  IF AND(param.载重 > 3000, param.速度 > 25) → 显示警告

规则3：电机功率下限 = 载重 × 速度 × 0.0115（动态范围约束）
  电机功率.min = param.载重 * param.速度 * 0.0115

规则4：选择"不锈钢"材质时，表面处理固定为"酸洗钝化"（联动规则）
  IF param.材质 == '不锈钢' → param.表面处理 = '酸洗钝化' (锁定)
```

**适用场景：** 设计规范强制、安全约束、工艺可行性检查

#### 场景9：参数继承 ⬇️

**核心问题：** 上级装配体的参数要自动传递到下级零件。

```
货架 (总高=2800, 层数=4, 载重=1000)
│
├─ 立柱-参数化 × 4
│  └─ 高度 = parent.总高         ← 从货架继承
│  └─ 载重 = parent.载重 / 2     ← 继承后运算
│
├─ 横梁 × 8
│  └─ 跨度 = parent.总宽 - 100   ← 从货架继承后运算
│
└─ 层板 × param.层数
   └─ 厚度 = IF(parent.载重 > 1000, 2.0, 1.5)
```

**继承规则：**
```
// 三级传递：上层 → 本层 → 下层
// 每层可以：
// 1. 原样传递：parent.参数名
// 2. 运算后传递：parent.参数名 + 偏移量
// 3. 覆盖：本层同名参数覆盖上级的
// 4. 阻断：标记不向下传递
```

#### 场景10：成本计算 💰

**核心问题：** 零件的成本和价格由参数决定。

**公式示例：**
```
# BOM项成本
单价公式: param.高度 * 0.035 + 25
// 每mm材料费0.035元 + 加工费25元

# 总成本 = 数量 × 单价
总成本: sys.quantity * (param.高度 * 0.035 + 25)

# 批量折扣
IF(sys.quantity > 100, 单价 * 0.9, 单价)   // 超过100个九折
IF(sys.quantity > 1000, 单价 * 0.8, 单价)  // 超过1000个八折

# 配置总成本汇总
配置总成本 = SUM(所有BOM项.数量 × 单价)
```

**输出：** 配置器实时显示总成本、成本构成明细、参数变化带来的成本差异

#### 场景11：零件属性 🏷️

**核心问题：** 零件的属性（重量、材质、颜色等）也随参数变化。

**公式示例：**
```
# 零件重量
重量(kg) = param.长度 * param.宽度 * 厚度 * 密度 / 1000000

# 表面处理编码
表面处理编码 = IF(param.环境 == '户外', 'HD01-户外粉', 'HD02-室内粉')

# 包装规格
包装数量 = IF(param.长度 < 1000, 100, 50)  // 小件100个一箱，大件50个

# 检验标准
检验标准 = IF(param.精度等级 == '精密', 'ISO-2768-m', 'ISO-2768-c')
```

**适用场景：** 仓储标签生成、质检标准动态选择、包装规格优化

#### 场景12：BOM结构动态变化 🏗️

**核心问题：** BOM的层级结构本身也会变化——有些子装配可能需要，有些不需要。

```
产品：可配置输送机

参数：速度=10, 载重=500, 有转弯=true

BOM结构展开：
┌─ 驱动段 × 1
│  ├─ 电机-0.75kW × 1
│  └─ 驱动辊道 × 2
├─ 中间段 × CEIL(param.长度/1000)
│  └─ 从动辊道 × CEIL(1000/500)
├─ 转弯段 × 1  ← 条件包含 (param.有转弯 == true)
│  └─ 转弯辊道 × CEIL(600/500)
│  └─ 转弯电机 × 1
└─ 机架 × 1
   ├─ 标准支座 × 4
   └─ 加强支座 × 0  ← IF(载重>2000, 2, 0)

参数：速度=10, 载重=500, 有转弯=false

BOM结构展开（不同结构）：
┌─ 驱动段 × 1
│  ├─ 电机-0.75kW × 1
│  └─ 驱动辊道 × 2
├─ 中间段 × CEIL(param.长度/1000)
│  └─ 从动辊道 × CEIL(1000/500)
└─ 机架 × 1                            ← 转弯段整个消失
   └─ 标准支座 × 4
```

**实现方式：** 子装配体也是一个参数化 Part，其条件公式控制它是否出现在BOM中。

```python
# 在 BOM 中引用子装配体时的条件
{
    "part": "转弯段_参数化模板",
    "condition_formula": "param.有转弯 == true",
    # 如果条件为 false → 整个转弯段及其下级BOM都不出现
}
```

### 15.4 场景组合矩阵

实际项目中，这些场景通常**组合使用**：

```
                  ┌── 场景1 (数量) ←──── 场景7 (计算参数)
                  │                          ↑
     ┌── 场景3 (选零件) ←──── 场景8 (参数约束)
     │
BOM ──┼── 场景4 (生成变体) ←─── 场景9 (参数继承)
     │
     ├── 场景5 (规格描述) ←─── 场景10 (成本计算)
     │
     └── 场景12 (结构变化) ←── 场景2 (条件包含)

产品配置器 UI 中：
  ┌─ 用户输入驱动参数 (长度、速度、载重)
  ├─ 场景7 自动计算出衍生参数 (电机功率、总重)
  ├─ 场景8 实时约束用户输入 (不允许越界组合)
  ├─ 场景1+2+12 确定BOM结构和数量
  ├─ 场景3+4+5 确定每个BOM项用什么零件
  └─ 场景10 实时显示成本变化
```

### 15.5 参数化场景速查手册

遇到新零件不知道用哪个场景时，按此流程自查：

```
┌─────────────────────────────────────────────────────────┐
│  参数化场景速查手册                                      │
│  用法：从上往下走，找到你的情况对应的场景号               │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  STEP 1: 这个零件有没有现成的 Part 编号？                │
│  │                                                        │
│  ├── ✅ 有（标准件/通用件，如螺栓、轴承、电机）          │
│  │   └── 每次用量是固定值吗？                            │
│  │       ├── 固定 → 标准BOM，不需要参数化                 │
│  │       └── 不固定（如长度不同数量不同）                  │
│  │          → 用【场景① 数量公式】                        │
│  │                                                          │
│  └── ❌ 没有（需要按单生产/采购）                          │
│      ├── 有标准型号系列（如立柱分几种标准高度）            │
│      │   ├── 型号少（≤20种）                              │
│      │   │  → 建好Part + 【场景③ 选件公式】                │
│      │   └── 型号多/连续变化（高度从1000~6000）            │
│      │     → 【场景④ 动态生成变体Part】                    │
│      │                                                      │
│      └── 完全定制，每单尺寸规格都不同                      │
│        → 【场景⑤ 规格描述，不创建Part】                     │
│                                                              │
│  STEP 2: 这个零件每个订单都出现吗？                        │
│  ├── 是 → 已通过STEP 1确定方案                              │
│  └── 不是 → 叠加【场景② 条件公式】控制是否出现              │
│                                                              │
│  STEP 3: 参数值来源是什么？                                │
│  ├── 用户直接填写 → 驱动参数                                │
│  ├── 由其他参数算出来 → 【场景⑦ 计算参数】                 │
│  ├── 从上级产品继承 → 【场景⑨ 参数继承】                   │
│  └── 需要在范围内受约束 → 【场景⑧ 参数约束】               │
│                                                              │
│  STEP 4: 还需要什么？                                      │
│  ├── 自动算成本 → 叠加【场景⑩ 成本计算】                   │
│  ├── 选供应商 → 叠加【场景⑥ 供应商选择】                   │
│  ├── BOM结构随参数变 → 【场景⑫ BOM结构】                   │
│  └── 零件属性随参数变 → 【场景⑪ 零件属性】                 │
└─────────────────────────────────────────────────────────┘
```
